Add the daily_logs table creation to init_db() in db.py, AFTER the existing items table creation. Schema:

```sql
CREATE TABLE IF NOT EXISTS daily_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    log_date TEXT NOT NULL,
    item_id INTEGER NOT NULL,
    dosage_taken REAL,
    notes TEXT NOT NULL DEFAULT '',
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (item_id) REFERENCES items(id),
    UNIQUE(log_date, item_id)
);
CREATE INDEX IF NOT EXISTS idx_daily_logs_date ON daily_logs(log_date);
CREATE INDEX IF NOT EXISTS idx_daily_logs_item ON daily_logs(item_id);
```

    GREEN phase -- implement services/log_service.py:

    - upsert_log_entry(log_date: str, item_id: int, dosage_taken: float | None, notes: str = "") -> None
      Uses INSERT OR REPLACE with ON CONFLICT(log_date, item_id) DO UPDATE.

    - take_all_fixed_dose(log_date: str) -> int
      INSERT OR IGNORE for all active items with default_dosage that don't already have a log for that date. Returns number of rows inserted.

    - get_logs_by_date(log_date: str) -> list[dict]
      SELECT from daily_logs JOIN items WHERE log_date = ? AND is_active = 1, ordered by sort_order, name. Returns list of dicts with keys: item_id, item_name, dosage_taken, notes, default_dosage, dosage_unit.

    - build_log_grid(target_date: str) -> pd.DataFrame
      Query all active items (ordered by sort_order, name). LEFT JOIN with daily_logs for target_date. Build a single-row DataFrame with item names as columns, dosage_taken as values, target_date as index. Items with no log entry get NaN. This ensures the grid always shows all active items even if no logs exist.
