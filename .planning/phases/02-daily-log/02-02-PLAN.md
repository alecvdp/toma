  <action>
Create pages/daily_log.py as the core daily logging interface. Follow established patterns from pages/catalog.py (import from services, not db).

**Page structure (top to bottom):**

1. **Title:** `st.title("Daily Log")`

2. **Date navigation (LOG-06):** `st.date_input("Date", value=date.today())` in a columns layout. Convert to string with strftime("%Y-%m-%d") for service calls.

3. **Take All button (LOG-04):** `st.button("Take All Defaults")`. On click, call `take_all_fixed_dose(log_date_str)` then `st.rerun()`. Show a success toast with count of items filled.

4. **Grid (LOG-01, LOG-02, LOG-03, CAT-10):**
   - Get active items via `get_active_items()` -- but sort by sort_order, name (not just name). Build item lookup dict {name: item_dict} for column config.
   - Call `build_log_grid(log_date_str)` to get the DataFrame.
   - Build `column_config` dict: for each item, create `st.column_config.NumberColumn` with help text showing "Default: {dosage} {unit}" if default_dosage exists, min_value=0.0, format="%.1f".
   - Render with `st.data_editor(grid_df, key="log_grid", on_change=handle_grid_edit, column_config=col_config, hide_index=False, num_rows="fixed", use_container_width=True)`.
   - The `column_order` param should list item names in sort_order sequence (from the DataFrame columns, which build_log_grid already orders).

5. **on_change callback (handle_grid_edit):**
   - Read `st.session_state["log_grid"]["edited_rows"]`.
   - For each `{row_idx: {item_name: new_value}}`, look up item_id from item lookup dict, call `upsert_log_entry(log_date_str, item_id, new_value)`.
   - The upsert is idempotent so re-processing accumulated edits is safe (per research pitfall #2).

6. **Notes section (LOG-05):**
   - Below the grid, show `st.subheader("Entry Notes")`.
   - Call `get_logs_by_date(log_date_str)` to get all entries for this date.
   - Filter to entries where dosage_taken is not None (items that have been logged).
   - For each logged entry, show the item name and a `st.text_input` for the note, keyed by item_id.
   - When user changes a note, call `upsert_log_entry(log_date_str, item_id, entry["dosage_taken"], notes=new_note)` to persist.

7. **Legend:** Small `st.caption` explaining: "Empty = not yet logged. 0 = skipped. Any number = dosage taken."

**app.py update:**
- Add Daily Log page to the navigation. It should appear as the FIRST page (primary workflow), with Catalog second.
- Add a `st.page_link` or use `st.navigation` / `st.Page` to include pages/daily_log.py.

**Important notes:**
- The item lookup for on_change callback needs item name -> item_id mapping. Build this from get_active_items() at page load. Store in a variable (not session state -- it's derived data).
- Do NOT use `hide_index=True` -- dates should show as row labels.
- The grid shows a single row per date (the selected date). This is intentional -- build_log_grid returns a 1-row DataFrame.
